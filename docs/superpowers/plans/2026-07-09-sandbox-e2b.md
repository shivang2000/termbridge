# P1.1 ‚Äî Live cloud SandboxProvider (E2B) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A concrete `@termbridge/sandbox-e2b` package implementing `SandboxProvider` against the E2B SDK, wired so `env: "sandbox"` is selectable end-to-end (SessionManager factory + MCP `open_session` enum), with a creds-gated live cloud smoke.

**Architecture:** The `SandboxProvider` interface + `SandboxEnvironment` class already exist in `@termbridge/core` (M6). This adds (1) a new `@termbridge/sandbox-e2b` package providing a concrete provider, (2) a `sandboxProvider` option on `SessionManagerOptions` so the default factory constructs `SandboxEnvironment` instead of throwing, (3) `"sandbox"` added to the MCP `open_session` enum, and (4) a creds-gated smoke. Core stays dependency-free (D3) ‚Äî only the new package imports the `e2b` SDK.

**Tech Stack:** Bun (test runner + workspaces) ¬∑ Turbo (pipeline) ¬∑ Biome (lint) ¬∑ TypeScript (NodeNext, strict) ¬∑ `e2b` SDK 2.32.0 ¬∑ zod. All unit tests are mocked (no real cloud); `bun run test` stays green with no creds.

**Spec:** `docs/superpowers/specs/2026-07-09-sandbox-e2b-design.md`

## Global Constraints

- **No `e2b` import in `@termbridge/core`** (D3 ‚Äî core stays zero-runtime-dependency). Only `@termbridge/sandbox-e2b` imports the SDK.
- **`SandboxProvider.exec` must NEVER reject on non-zero exit** ‚Äî returns `{ stdout, stderr, code }` verbatim (the `ExecFn` contract `DockerEnvironment` + tmux helpers honour). The E2B SDK throws `CommandExitError` on non-zero; the provider catches it and extracts the fields.
- **All unit tests mocked** ‚Äî never spawn a real cloud or real tmux. The provider takes an injectable `sandboxFactory`; manager/tools tests use a fake `Environment`.
- **`stdio` stays the zero-infra default** ‚Äî sandbox is opt-in.
- **Tab indentation** (Biome enforces; the repo uses tabs). **`.js` extensions on relative imports** (NodeNext + `verbatimModuleSyntax`).
- **Test framework:** `bun:test` (`import { describe, expect, test } from "bun:test"`).

- [ ] **Step 3: Implement `packages/sandbox-e2b/src/e2b-provider.ts`**

The full implementation is provided as `docs/superpowers/plans/_snippets/e2b-provider.ts` (written to disk) ‚Äî copy it verbatim to `packages/sandbox-e2b/src/e2b-provider.ts`. Key points it implements:
- `import { Sandbox } from "e2b"` + `import type { ExecResult, SandboxProvider } from "@termbridge/core"` (the only `e2b` import in the whole repo; core stays dep-free).
- `E2BSandboxProviderOptions`: `apiKey?` (defaults to `E2B_API_KEY` env), `template?` (default `"base"`), `timeoutMs?` (default `3_600_000`), `sandboxFactory?` (injectable; defaults to `Sandbox.create`).
- `ensure(opts)`: calls `sandboxFactory({ template: opts.image ?? this.template, envs, apiKey, timeoutMs, metadata: { name } })`, stores the sandbox, then runs an idempotent tmux install probe (`command -v tmux || apt-get install -y tmux`) since `base` lacks tmux.
- `exec(args)`: `execRaw(args.map(shellQuote).join(" "))` where `shellQuote` single-quote-wraps args with spaces/metachars.
- `execRaw(cmd)`: `sandbox.commands.run(cmd, { timeoutMs })`; on throw, if the error has `exitCode` (the `CommandExitError` shape) return `{ stdout, stderr, code: exitCode }` as data; else rethrow (genuine SDK error).
- `destroy()`: clears the stored sandbox; if one exists, `sb.kill()` wrapped in try/catch (never throws); no-op if `ensure` was never called.

# Sandbox (cloud) integration ‚Äî E2B

termbridge's `env:"sandbox"` runs a session's tmux INSIDE a cloud sandbox, behind
the same `Environment` interface as `local`/`docker` (D4). A concrete provider ships
in `@termbridge/sandbox-e2b` (E2B); Daytona/Cloudflare are future ports behind the
same `SandboxProvider` interface.

## Setup

1. Get an E2B API key (`E2B_API_KEY`). The default `base` template is used; tmux is
   installed on first `ensure` (it isn't pre-baked into `base`).

2. Wire the provider into the `SessionManager`:

   ```ts
   import { SessionManager } from "@termbridge/core";
   import { E2BSandboxProvider } from "@termbridge/sandbox-e2b";

   const manager = new SessionManager({
     sandboxProvider: new E2BSandboxProvider({ apiKey: process.env.E2B_API_KEY }),
   });
   const session = await manager.open({ env: "sandbox", cwd: "/root", cmd: "claude" });
   ```

3. Over MCP, `open_session` now accepts `env: "sandbox"`. If the server's manager
   has no `sandboxProvider` configured, it throws `sandbox_not_configured`.

## Live smoke (creds-gated)

```bash
E2B_API_KEY=... bun scripts/smoke-sandbox-e2b.ts
```

Without `E2B_API_KEY` the smoke no-ops and exits 0 (so `bun run test` stays green).
```
